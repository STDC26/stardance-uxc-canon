SD Unified Experience Contract · v1.2
The contract every surface
must conform to
The UXC governs how every surface in the Stardance platform looks, feels, and behaves. It is the document UAT tests against. Without it, there is no canonical definition of done. With it, every surface is accountable to the same seven contract layers, the same Design Doctrine, and the same five visibility requirements.
The Five Visibility Requirements
Every operator-facing surface must expose all five. If any answer is invisible, the surface fails the UXC.
C0 · The Human Principle
01
What signals were used
Which data inputs drove this output. Source and recency visible.
02
What canon was applied
Which Canon assets governed the decision. HCTS version, PLA version.
03
What decision was made
The verdict. PLA band. State. Not implied — explicit.
04
What confidence exists
Evidence quality score. FLAG-B01. Band. Cap acknowledged.
05
What will be learned
What outcome signal this execution will generate for EDGE.
C4 — The Design Doctrine · Governing all surfaces
Clarity → Flow → Delight
Every surface the platform builds is subordinate to this sequence. Not aesthetic preference — sequenced execution doctrine.
First
Clarity
The message must be immediately legible before any other quality matters. If the user cannot understand what they are looking at in three seconds, the surface has failed the first gate.
→
Second
Flow
Friction removed from the path to action. Every interaction decision is measured against whether it accelerates or impedes the operator's intended action.
→
Third
Delight
Only once clarity and flow are achieved does emotional engagement compound conversion. Delight that exists without clarity or flow is decoration, not experience.
IMS — Interaction Model Standard
IMS
Required
Interaction Model Standard
Canonical execution state machine · application-layer flows only
Defines the single canonical execution model for operator-facing and LLM-driven flows. Eliminates behavioral drift by ensuring every system reports execution state in the same vocabulary.
idle → validating → processing → complete · partial_complete · failed
Scope: operator_initiated, LLM_driven, surface_level workflows. Does NOT replace FCS subsystem states (active/paused/degraded/blocked) or domain states (CIF ReviewState, CORTEX governance)
Every state must expose: state · confidence · validation_status · next_action
Invalid transition → Block · Missing state → Block · Missing confidence → Block

LLM Behavior Contract
Canonical output structure · prompt governance · structure enforcement
Governs how LLM outputs are structured for operator-facing surfaces. Prevents LLM behavior drift by requiring canonical three-part output structure on all new builds. Existing outputs under grace-period migration.
All new LLM outputs must structure: what_happened · why_it_matters · what_to_do_next

State Visibility Standard
Execution state · confidence · validation — always visible
Governs how execution state and confidence are surfaced to the operator. No state may be hidden, inferred, or presented ambiguously. The Five Visibility Requirements are the SVS checklist.
State must be visible before, during, and after every execution

next_action must always be present — operator must never be left in ambiguity
Hidden state = SVS violation. Implied state = SVS violation.

TIS
Required
Trust & Insight Surface
Confidence · validation · trust signals — never implied, never omitted
Governs how trust and confidence are communicated to the operator. Every surface that presents an intelligence output must make the evidence quality explicit. No trust signal may be implied by absence, design treatment, or default state. TIS is C0 (The Human Principle) operationalised as a display contract.
Confidence display
FLAG-B01 score + band (HIGH/MEDIUM/LOW) must be visible. The 0.92 cap must be acknowledged — no surface may imply 100% certainty.
Validation status
Block / Flag / Note must be surfaced with rationale. A Block may never be silenced by UI treatment. All governance decisions must be traceable.
Ethics gate visibility
If an Ethics Gate (C6) block is triggered, the operator must see which category (VS-001/002/003 or PCC-001/002/003/004) triggered it. Not just "blocked" — why blocked.
